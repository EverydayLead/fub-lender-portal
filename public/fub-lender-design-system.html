Accessibility

Banking apps serve diverse users across Community Bank locations. These standards ensure WCAG 2.1 AA compliance across all components.

Contrast Ratios

Body text (14px): minimum 4.5:1 against background. Large text (18px+ or 14px bold): minimum 3:1. All interactive elements: minimum 3:1 against adjacent colors.

#002D72 on white10.5:1 ✓
#54585A on #FAF9F55.7:1 ✓
#78BE21 on white2.8:1 — large text only

Touch Targets

All interactive elements (buttons, links, form fields, cards, nav items) maintain a minimum touch target of 44×44pt. Padding or invisible hit areas extend small visual elements to meet this minimum.

Screen Reader Labels

Every component exposes an accessibilityLabel prop. StatusBadge reads as "Status: In Review." SLACountdown reads as "SLA deadline: 3 hours 42 minutes remaining." RiskCard reads as "Risk level: Low, score 82 out of 100."

Color-Blind Safety

Status is never conveyed by color alone. StatusBadge always includes text labels. RiskCard uses distinct geometric icons (✓, ◆, ▲, ✕). SLACountdown includes text description alongside color. LaneBadge includes icon differentiation (⚡, ◆, ⚠).

Reduced Motion

When prefers-reduced-motion: reduce is active, all animations complete instantly. SLA pulse stops. Skeleton loading shows a static gray block. Screen transitions cross-fade in 0ms.

Focus Management

Keyboard/switch-control focus uses a 2px blue outline offset by 2px. Focus order follows visual layout top-to-bottom, left-to-right. Modals trap focus. Approval actions require explicit confirmation — no accidental activations.
